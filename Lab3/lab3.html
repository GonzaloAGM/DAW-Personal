Descripción

En esta actividad se hará un repaso sobre CSS.

Objetivos:

  • Comprender el funcionamiento de las hojas de estilo en cascada.
  • Aplicar los 3 niveles de estilos (en línea, documento y externo)
  • Aplicar los diferentes tipos de selectores (genéricos, de clase, id, universales y pseudo clases)
  • Explorar las propiedades de fuentes, listas y texto
  • Comprender el modelo de caja (The Box Model)
  • Aprender a separar la presentación del contenido en documentos HTML.

Instrucciones

  1. Revisa la presentación CSS.
  2. El CSS debe contener al menos un estilo definido con cada uno de los selectores y el estilo debe ser aplicado al documento. Puedes utilizar como guía o o ayudarte de algunos de los sitios web que se enlistan en la sección de Recursos, o en algún otro sitio.
    Asegúrate que tus documentos HTML no tengan estilos en línea.
  3. Para un mejor rendimiento de tu sitio, crea una versión minimizada de tu CSS y enlázala a tu sitio en lugar del CSS que acabas de crear. Puedes apoyarte de herramientas como http://cssminifier.com/
  4. Agrega las preguntas con sus respectivas respuestas en el documento HTML.

Preguntas a responder

  1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    2. Utilizarlo solo cuando sea necesario, por ejemplo, que solo arreglas el problema que tienes con eso. O cuando sabes perfectamnte que es lo que haces y que no afectará nada mas.

  2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    3. Por la resolucion de la imagen, y por consecuente, las pantallas en donde se estara viendo. Por la accesibilidad al usuario, se deben considerar si es facil de leer el contenido.

  3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    4. Creo que cada una tiene su lugar para utilizarse excepto px. Quizas solo para imagenes o bordes. En cuanto a % y pt, yo usaria el porcentage para pocisionar algo relativo a otro elemento o la pantalla, y pt para algo estatico y texto.

  4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    5. Al no tener saltos de linea, ni espacios de mas, que aunque útiles para identación, ya no se consumirían tantos recursos en memoria del servidor que aloja la página web y trasnporte a través del internet.
